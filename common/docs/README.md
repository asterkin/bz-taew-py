This documentation provides a comprehensive overview of the **BlueZone** project, including its use cases, architectural decisions, and key concepts.

- **[Glossary](./glossary.md)**: Definitions of key terms and concepts used throughout the project.
- **[Use Cases](./use-cases/README.md)**: Detailed specifications of the system's supported use cases.
- **[Architecture Decision Records (ADRs)](./adrs/README.md)**: Documentation of key architectural decisions made during development.
- **[Acceptance Tests](./features/README.md)**: A language- and technology-neutral acceptance test suite, derived from the [use cases](./use-cases/README.md), designed to verify consistent application behavior across different technology stacks.

## How to Use This Documentation

- Begin with the **Glossary** to understand the core terminology.
- Review the **Use Cases** to explore the system's supported scenarios.
- Dive into the **Acceptance Tests** to see how expected behaviors are validated.
- Explore the **ADRs** to learn about the project's architectural decisions and their rationale.

For a high-level introduction and project overview, refer to the [top-level README](../README.md).
